# Requirements Document

## Introduction

区域多站点雨量数据质量控制系统（V1），对区域内多个气象/水文站点的降水数据进行自动化质量检查与异常标记。目标是揪出时间序列中不符合物理规律或气候统计的异常片段。系统从时间序列单一维度（单站时序规则）和空间一致性维度（多站横向比对）两个维度进行质量控制。

数据按站点目录组织（如 `RegionalStation西安/`、`RegionalStation武汉/`），每个站点目录下存放该站的全部数据表文件。检测以站点为单位执行——选择目标站点后，系统加载同区域所有站点数据进行跨站比对，仅输出目标站点的检测结果。每个站点生成一个同名 Excel 报告（如 `西安.xlsx`），按表类型分 Sheet 输出。

V1 阶段站点为手动录入，不需要 GIS 半径自动搜邻站。

## Glossary

- **站点（Station）**：气象/水文观测站点，每个站点对应一个目录（如 `RegionalStation西安/`），目录下存放该站全部数据表文件，站点元数据（经纬度、高程等）记录在站点目录下的 `station.yaml` 中
- **降雨摘录表（Rainfall Excerpt Table）**：按时间段（小时/时段）记录的降雨摘录数据，每行包含站点、时间、降雨量
- **逐日降水表（Daily Precipitation Table）**：按日记录的降水量，带"*"表示当日降雪
- **月年降水对照表（Monthly/Yearly Comparison Table）**：按月/年汇总的降水量、降水日数对照数据
- **各时段最大降水量表（Max Rainfall by Period Table）**：按各时段（1h、3h、6h、12h、24h 等）统计的最大降水量
- **汛期（Flood Season）**：5、6、7、8、9、10 月
- **非汛期（Non-Flood Season）**：1、2、3、4、11、12 月
- **质量标记级别**：提示（Info） < 一般（General） < 警告（Warning） < 严重（Severe）
- **持续性微量降雨（Persistent Trace Rainfall）**：因树叶、冰冻、昆虫等导致翻斗式雨量筒持续检出微小值（0.1 / 0.2 mm）
- **僵直值（Stagnation Value）**：数据长时间保持相同非零值，翻斗卡住的典型特征
- **跳变/阶跃（Jump / Step）**：短时间内雨量剧增后立刻归零，仪器震动或数据传输错误
- **空间一致性比率 Q**：Q = 本站降雨量 / 邻站平均降雨量，衡量单站与区域均值的偏离程度
- **反距离加权（IDW）**：以距离倒数加权的空间插值方法，用于估计本站期望雨量
- **克里金残差（Kriging Residual）**：基于克里金插值的观测值与估计值之差
- **MAD（Median Absolute Deviation）**：中位数绝对偏差，用于稳健的离群值检测
- **阈值**：系统中所有检测阈值均支持外部配置文件修改

## References

- TADA (USGS): 水环境数据质量控制 Python 库，含异常检测、平滑、可视化
- GSODR / GHCN-Daily: NOAA NCEI 全球历史气候学网质量标记系统（M 标记），算法公开
- climatol (R): 气候序列均一性检验和空间一致性检验
- pykrige / scikit-gstat: Python 克里金插值库
- PyOD / adtk: Python 时间序列异常检测库（孤立森林、LOF 等无监督方法可作辅助）

## Requirements

### Requirement 1: 持续性微量降雨检测

**User Story:** AS a 数据质量控制人员, I want 系统自动识别因树叶/冰冻等导致翻斗雨量筒持续检出微小值的异常片段, so that 消除伪降雨记录

#### Acceptance Criteria

1. 系统 SHALL 使用滑动窗口算法对每个站点的小时尺度雨量序列逐窗口扫描
2. WHEN 连续 N 个小时（默认 N=12，可配置）雨量均等于 0.1 或 0.2 mm 且期间无一次大于 0.5 mm 的有效降雨，the system SHALL 标记该窗口内数据为"持续性微量降雨"异常
3. 阈值 0.1、0.2、0.5、N SHALL 支持配置文件修改
4. 系统 SHALL 在输出结果中标注异常窗口起始时间、持续时长、最大雨量值

### Requirement 2: 僵直值检测

**User Story:** AS a 数据质量控制人员, I want 系统检测雨量值长时间无变化的异常片段, so that 识别翻斗卡住等设备故障

#### Acceptance Criteria

1. WHEN 单站连续超过 H 个小时（默认 H=24，可配置）雨量为非零恒定值，the system SHALL 标记该段为"僵直值"异常
2. 系统 SHALL 区分"持续零值"与"僵直非零值"——持续零值可能是无雨期，属于正常；仅标记持续恒定非零值
3. 僵直持续时长阈值 H 和容许波动容差（默认 0 mm）SHALL 支持配置

### Requirement 3: 气候极值检查

**User Story:** AS a 数据质量控制人员, I want 系统检查降雨量是否超出气候学合理范围, so that 快速过滤明显异常的大值

#### Acceptance Criteria

1. WHEN 时雨量大于配置的当地历史最大小时雨量（默认 150 mm，可配置），the system SHALL 标记为"超气候极值"警告
2. WHEN 日雨量大于配置的当地历史最大日雨量（可配置），the system SHALL 标记为"超气候极值"警告
3. 系统 SHALL 支持基于百分位数的动态阈值——计算历史序列 P99.9 分位数，对超过该阈值的日雨量标记为"极端值"提示，建议人工复核
4. 极值阈值 SHALL 支持按站点分别配置（不同站点气候背景不同）

### Requirement 4: 突跳/阶跃检测

**User Story:** AS a 数据质量控制人员, I want 系统检测短时雨量突变后立刻归零的异常, so that 排除仪器震动或数据传输错误

#### Acceptance Criteria

1. 系统 SHALL 使用基于滑动窗口的中位数绝对偏差（MAD）法——对每个站点的小时雨量序列，计算滑动窗口内的中位数和 MAD，当某值偏离窗口中位数超过 N 倍 MAD（默认 N=5，可配置）时标记
2. 系统 SHALL 同时计算前后相邻值的变化率——当 1 小时内雨量激增超过配置倍率阈值且随后归零时标记为"阶跃"异常
3. 系统 SHALL 根据突变幅度分档输出不同告警级别——突变倍率（跳变值/前值）越大，告警级别越高（如 2~5 倍提示、5~10 倍一般、10~30 倍警告、>30 倍严重，各级阈值可配置）
4. MAD 窗口大小、偏差倍数、变化率分档阈值 SHALL 支持配置

### Requirement 5: 降雨摘录表跨站横向对比

**User Story:** AS a 数据质量控制人员, I want 系统对比区域内所有站点同一时间段的降雨量, so that 通过空间一致性识别孤立异常站点

#### Acceptance Criteria

1. WHEN 系统接收降雨摘录表数据，the system SHALL 对每个时间段计算所有站点的雨量均值和标准差，标记偏离均值超过 N 倍标准差（默认 N=3，可配置）的站点
2. 系统 SHALL 输出异常站点的站名、时段、实测值、区域均值、偏差值、触发规则

### Requirement 6: 检测执行模式

**User Story:** AS a 数据质量控制人员, I want 系统以站点为单位执行检测, so that 每个站点独立输出同名 Excel 报告

#### Acceptance Criteria

1. 系统 SHALL 以站点为检测单位——用户选择目标站点，系统加载同区域目录下所有站点的数据执行跨站比对，仅输出目标站点的异常结果
2. 目标站点的单站时序检测器（持续性微量、僵直、极值、突跳、人为错误）仅对该站数据执行
3. 跨站检测器（横向对比、零雨、空间一致性、月/年对照）使用区域全部站点的数据做比对，但仅在目标站点触发异常时写入结果
4. 所有检测模式 SHALL 仅输出质量标记结果报告，不修改原始数据
5. 系统 SHALL 对同一站点多次检测生成独立报告，不覆盖历史报告

### Requirement 7: 各时段最大降水量表检查

**User Story:** AS a 数据质量控制人员, I want 系统检查各时段最大降水量表中数据的内部一致性, so that 确保时段统计汇总结果无误

#### Acceptance Criteria

1. WHEN 系统接收各时段最大降水量表数据，the system SHALL 检查连续各时段值是否合理——不能出现短时段（如 1h）最大值大于长时段（如 6h）最大值的逻辑矛盾
2. 系统 SHALL 对各时段数据执行跨站横向对比，计算 Q 值（本站/邻站均值），标记 Q 超出配置阈值范围的站点
3. 系统 SHALL 执行僵直值检测——同一站点各时段最大值完全相同的标记异常

### Requirement 8: 逐日降水表—邻近站点零雨检测

**User Story:** AS a 数据质量控制人员, I want 系统对本站有雨而区域内所有邻站无雨的情况进行检测, so that 识别伪降雨

#### Acceptance Criteria

1. WHEN 本站某日雨量 P > 0 且区域内其余所有站点该日雨量均为 0（或均小于 0.2 mm）且本站雨量小于 2 mm，the system SHALL 标记为"疑似伪降雨"，级别为警告
2. 系统 SHALL 对降雪日（带"*"）采用相同检测逻辑，微小阈值调整为 0.3 mm（可配置）
3. 雨量阈值 0.2、2.0 SHALL 支持配置

### Requirement 9: 逐日降水表—空间一致性比率检测

**User Story:** AS a 数据质量控制人员, I want 系统通过空间一致性比率 Q 检测站点降雨量相对区域的异常偏离, so that 区分真正的局地强对流和仪器误差

#### Acceptance Criteria

1. 系统 SHALL 对每日每站计算 Q = 本站日雨量 / 区域内其他站点平均日雨量
2. WHEN 邻站均值大于 0 且 Q 大于配置阈值（默认 3~5，可配置）且本站值不大（小于 5 mm），the system SHALL 标记为"疑似伪雨（树叶干扰场景）"，级别为警告
3. WHEN 邻站均值大于 0 且 Q 大于配置阈值且本站值较大（≥ 5 mm），the system SHALL 标记为"可能局地强对流，建议结合雷达复核"，级别为提示
4. WHEN 邻站均值大于 0 且 Q 小于配置下限（默认 0.05，可配置，即本站异常偏小），the system SHALL 标记为"本站异常偏小"，级别为一般
5. 空间插值方法（IDW 或克里金）作为进阶检查——计算本站期望雨量，当实测值与期望值标准化残差大于 3 倍标准差时标记——V1 阶段作为可选功能

### Requirement 10: 逐日降水表—人为错误检测

**User Story:** AS a 数据质量控制人员, I want 系统检测降水数据中的系统性人为录入偏差, so that 消除观测员习惯性取整/取偶等错误

#### Acceptance Criteria

1. WHEN 数据来源于非汛期人工观测，the system SHALL 检查逐日降水量——若有效降水日（日雨量 > 0）全部为偶数则标记为"疑似人工取偶"异常，级别为一般
2. WHEN 数据来源于汛期自动雨量站，the system SHALL 检查逐日降水量——出现奇数则标记为"自动站不应有奇数"异常，级别为一般
3. 系统 SHALL 根据配置文件中的站点属性（观测方式：人工/自动）和当前日期自动切换检查规则

### Requirement 11: 月降水对照表—汛期/非汛期差异化检查

**User Story:** AS a 数据质量控制人员, I want 系统按汛期和非汛期采用不同策略检查月降水数据, so that 适应不同季节降水特征

#### Acceptance Criteria

1. WHEN 系统处理非汛期（1、2、3、4、11、12 月）月降水数据，the system SHALL 检查区域各站月降水量——非汛期各站差值应较小、倍数关系应明显，标记偏离此规律的站点
2. WHEN 系统处理汛期（5、6、7、8、9、10 月）月降水数据，the system SHALL 检查区域各站月降水量——汛期各站差值可大但不可无限大、倍数关系可不明显，标记偏离此规律的站点
3. 系统 SHALL 支持配置汛期/非汛期的月份划分

### Requirement 12: 年降水与年降水日数联合检查

**User Story:** AS a 数据质量控制人员, I want 系统联合检查年降水量和年降水日数的倍数关系, so that 从多维度发现异常站点

#### Acceptance Criteria

1. WHEN 系统对区域各站年降水量和年降水日数进行倍数关系分析，the system SHALL 区分以下情况：
   - 若年降水量与年降水日数两组数据的倍数关系均明显且差距过大：输出提示级别
   - 若仅一方倍数关系明显而另一方不明显：输出警告级别
2. 倍数关系判定阈值 SHALL 支持配置

### Requirement 13: 配置管理

**User Story:** AS a 系统运维人员, I want 所有检测阈值可通过配置文件统一管理, so that 适配不同地区、不同季节的降水特征

#### Acceptance Criteria

1. 系统 SHALL 使用 YAML 配置文件管理所有检测阈值，包括：
   - 持续性微量降雨：连续时长 N、低值范围、有效降雨阈值
   - 僵直值：持续时长 H、容许波动容差
   - 气候极值：时雨量上限、日雨量上限、百分位数阈值
   - 突跳/阶跃：MAD 窗口大小、偏差倍数、变化率阈值
   - 空间一致性：Q 值上下限、标准差倍数
   - 零雨检测：微雨阈值、伪降雨判定阈值
   - 人为错误：汛期/非汛期月份划分
   - 质量标记：各级别的偏差分级阈值
2. WHEN 配置文件不存在，the system SHALL 使用内置默认值运行并输出提示
3. 系统 SHALL 支持按站点独立配置极值阈值

### Requirement 14: 质量标记与报告输出

**User Story:** AS a 数据质量控制人员, I want 系统按站点输出同名 Excel 报告, so that 一个站一个文件便于归档和分发

#### Acceptance Criteria

1. 系统 SHALL 对每条异常按四级输出：提示（Info）、一般（General）、警告（Warning）、严重（Severe）
2. 每条异常记录 SHALL 包含：站点名称、日期/时段、异常值、期望值或参考值、偏差量、触发规则名称、标记级别
3. 系统 SHALL 为每个站点生成一个与该站点目录完全同名的 Excel 文件（如目录为 `RegionalStation西安/`，则输出 `RegionalStation西安.xlsx`），文件内含多个 Sheet：首个 Sheet 为汇总统计，后续按数据表类型分 Sheet（"降雨摘录表"、"逐日降水表"、"月年降水对照表"、"各时段最大降水量"），仅在该表有检出异常时出现对应 Sheet
4. 汇总统计 Sheet SHALL 包含按检测器和标记级别统计的异常数量分布表
5. 系统 SHALL 支持从 Web 界面下载单个站点的 Excel 报告

### Requirement 15: Web 界面

**User Story:** AS a 数据质量控制人员, I want 通过 Web 界面管理站点数据目录、配置阈值、执行检测并下载报告, so that 无需命令行操作即可完成质量控制全流程

#### Acceptance Criteria

1. 系统 SHALL 提供 Web 界面创建和管理站点目录——每个站点目录（如 `RegionalStation西安/`）下上传该站的降雨摘录表、逐日降水表、月年降水对照表、各时段最大降水量表文件（CSV / Excel）
2. 站点元数据（站名、经纬度、高程、观测方式）SHALL 自动从站点目录下的 `station.yaml` 读取，Web 界面可编辑
3. 系统 SHALL 提供 Web 界面选择目标站点执行检测——系统自动加载同区域其他站点数据用于跨站比对
4. 系统 SHALL 提供 Web 界面在线编辑检测阈值配置
5. 系统 SHALL 在 Web 界面展示质量检查结果——异常列表（支持筛选、排序）和汇总统计
6. 系统 SHALL 支持从 Web 界面下载单个站点的 Excel 报告（`站点目录名.xlsx`）
